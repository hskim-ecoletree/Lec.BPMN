# 새 구성안 — 단원 구조와 장표 목록 (사용자 리뷰용)

> COURSE.md §9에 따른 재구성안. **승인 전에는 제작하지 않는다.**
> 승인되면 이 문서가 구 설계안(course-plan.md)을 대체하는 제작 기준이 되고, 전 장표를 `docs/inventory.md`에 "계획" 상태로 등록한다.

## 설계 원칙 (헌장에서 도출)

1. **여섯 질문(Q1~Q6)이 커리큘럼의 뼈대다.** 오전은 질문 프레임과 언어(Q1·Q2), 오후는 분기·대기·복구(Q3·Q4·Q5), 마지막은 적합성(Q6). 모든 장표는 복무 질문이 있다.
2. **온보딩 이야기가 하루를 관통한다.** 각 교시의 첫 장표가 스토리의 다음 장면을 연다. 완성 모델은 2교시에 한 번 훑고, 이후 장면마다 요소를 발견한다.
3. **1장 1주제, 저밀도.** 구 설계안에서 한 장표에 눌러 담았던 소주제(예: Start 4종+설정표)를 나누거나 덜어냈다. 장당 4~7분.
4. **화면에는 강의 내용만.** 교시·시간은 이 문서와 aside.notes에만 존재한다. 교시 경계는 "회수 장표"(내용 제목)가 담당한다.
5. **기존 통과분 6장 재활용** (표지·OutSystems 비교·전체 이야기·토큰·교착·Async). 신규 제작 53장.
6. **사실은 verification-notes 것만.** Event-based Gateway 장표는 "Camunda 7은 Intermediate Catch Event만" 교정을 반영해 설계했다.

## 구 설계안 대비 달라진 것

- **다섯 질문 → 여섯 질문**: 도구 적합성(Q6)을 1교시부터 도입하고 8교시에서 닫는다 (구 설계안은 마지막에만).
- **토큰을 4교시 첫머리로 이동** (구: 6단원): 분기를 배우기 전에 세는 도구부터.
- **Boundary Event·채용 취소(Event Subprocess)를 6교시 복구 맥락으로 이동** (구: 요소 단원에 분산). 예외는 "요소 문법"이 아니라 "복구 설계"로 가르친다.
- **Modeler Properties 단원(구 19·21~29의 설정표) 폐지**: 설정값 나열 대신 각 개념 장표에서 실행 의미를 바꾸는 값만 그때그때. "Properties를 외우지 않는다"는 구 설계안 §4의 선언을 구조로 실현.
- **리뷰 체크리스트(구 17)를 실습 평가(55)로 이동**: 배울 때가 아니라 쓸 때 준다.
- 검증 교정 반영: Event-based Gateway(37) = Camunda 7은 Intermediate Catch Event만.
