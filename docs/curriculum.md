# 새 구성안 — 단원 구조와 장표 목록 (v2)

> COURSE.md §9에 따른 재구성안. **승인 전에는 제작하지 않는다.**
> 승인되면 이 문서가 제작 기준이 되고(구 설계안 course-plan.md 대체), 전 장표를 `docs/inventory.md`에 "계획" 상태로 등록한다.

## v2 반영 사항 (사용자 결정)

- 분량 **59 → 50장**: 병합 — 스토리 개막+흐름 구조, 부침 역사+남은 가치, Start+End Event, XOR 분기+합류, Parallel 분기+합류, Wait State+Timer, Definition/Instance+Cockpit. 삭제 — 해상도(대화·분석·실행) 단독 장표(8교시 판단 장표에서 언급), Sequence Flow 단독 장표(XOR 조건 설명에 흡수).
- 실습은 **집중형(7교시)** 유지.
- 제목을 **담백하게**: 명사형 중심, 수사 제거. 은유·구호·극적 표현·숫자 장치를 쓰지 않는다. 재활용 장표도 예외 없이 맞춘다.
- 용어: 시각 표현은 **"다이어그램"**으로 통일한다("도형"은 개별 모양을 가리킬 때만).

## 설계 원칙 (헌장에서 도출)

1. **여섯 질문(Q1~Q6)이 뼈대.** 오전 = 프레임과 언어(Q1·Q2), 오후 = 분기·대기·복구(Q3·Q4·Q5), 마지막 = 적합성(Q6).
2. **온보딩 이야기가 하루를 관통.** 각 교시 첫 장표가 다음 장면을 연다. 완성 모델은 2교시에 한 번 훑고, 이후 장면마다 요소를 발견.
3. **1장 1주제.** 병합 장표는 fragment 단계 노출로 분기→합류를 순서대로 다룬다(한 화면에 두 주제를 동시에 띄우지 않음).
4. **화면에는 강의 내용만.** 교시·시간은 이 문서와 aside.notes에만.
5. **기존 통과분 6장 재활용**, 신규 44장.
6. **사실은 verification-notes 것만.** Event-based Gateway는 "Camunda 7 = Intermediate Catch Event만" 교정 반영.
7. **Modeler Properties 설정표 단원 폐지** — 실행 의미를 바꾸는 값만 해당 개념 장표에서.
