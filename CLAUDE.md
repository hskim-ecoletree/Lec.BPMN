# Lec.BPMN — 작업 규칙

BPMN·Camunda 7 강의 슬라이드(reveal.js + bpmn-js). 원본은 HTML이며 PDF/PPTX는 export 산출물이다.

## 규칙의 원천 (우선순위 순)

1. **`COURSE.md`** — 과정 헌장. 목적(여섯 질문)·대상·비목표·배제 사항·핵심 메시지·용어 사전. **과정 수준의 최상위 기준** — 아래 문서·산출물이 헌장과 충돌하면 헌장이 이긴다.
2. **`DESIGN.md`** — 시각 규칙(색·타이포·컴포넌트) + **Slide Content Principles**(전달 원칙 11개 + 자가 점검 체크리스트 10문항). 장표·대본의 완성 기준.
3. **`docs/verification-notes.md`** — OutSystems·Camunda 7 사실 검증 기록. **여기 없는 사실 주장은 화면·대본에 올리지 않는다.** 어긋나는 내용 발견 시 먼저 공식 문서로 재검증 후 노트를 갱신한다.
4. **`docs/course-plan.md`** — 구 52장 설계안. **정본 아님, 참고 초안** — 재구성(갈아엎기) 시 문구·구성을 그대로 옮기지 않는다 (COURSE.md §9).

**`docs/inventory.md`** — 장표 등록부. 모든 장표는 여기 등록하고 복무 질문(Q1~Q6)을 명시한다. 등록 없는 장표는 없는 것으로 취급.

## 작업 흐름

- **장표 제작·수정** → `slide-author` 스킬을 먼저 로드하고 그 절차를 따른다. 완료 시 inventory 상태 갱신.
- **강사 대본 작성** → `script-author` 스킬을 먼저 로드한다. 대본은 각 `<section>`의 `<aside class="notes">`가 정본.
- **장표 완성 판정** → 브라우저 검증(메인 세션) 후 `slide-reviewer` 서브에이전트로 체크리스트를 판정받는다. FAIL 항목을 고치기 전에는 완료라고 말하지 않는다.
- **단원 완성·재구성 확정 등 마일스톤** → `deck-reviewer` 서브에이전트로 덱 전체 일관성(용어·금지 표현·트레이싱·회수)을 판정받는다.

## 검증 명령

```bash
python3 -m http.server 8753          # 정적 서버 (file://로는 bpmn fetch 불가)
# http://localhost:8753/index.html?nocache=N#/h  — 브라우저 캐시 우회는 nocache 값 증가 + force reload
# ?debug 붙이면 deck.js step 로그 출력
```

브라우저에서: 콘솔 오류 0, fragment 끝까지 진행하며 토큰 재생 확인, 스크린샷으로 겹침·과밀 확인.

## 불변 규칙 (요약 — 정본은 DESIGN.md v3)

- **디자인 v3 확정**: 표지=어둠 속 수직 광선, 본문=따뜻한 종이+좌측 그림자. 위계: **메시지(`.slide__msg`, 최상단·최대) > 도식(증거) > 구조 요소 > 배경**. 이 아이덴티티를 임의로 바꾸지 않는다 — 변경은 사용자 승인 필요.
- 색은 의미색 4종만: 강조·토큰 `--accent #457075`(사용자 확정) / 오류 테라코타 / 성공 녹색 / 대기 황토. `:root` 토큰만 사용.
- 화면 텍스트 30~60단어. 대사·설명은 `aside.notes`([다음] 동기화)로.
- 이모지·픽토그램·무의미한 장식(바·배지·칩) 금지.
- 한글 `word-break: keep-all` 전제 — 단어 중간 줄바꿈이 보이면 버그.
- theme.css/deck.js 수정 시 index.html의 `?v=` 캐시 버전을 올린다.
- 대본·화면에서 금지 표현: "전문가", "완전 정복", "실행 엔진 숙달", "Deep-Dive".
