## Overview

**Lec.BPMN**은 BPMN·Camunda 7을 "Workflow를 구조적으로 생각하는 도구"로 가르치는 전일(8교시) **강의 슬라이드 덱**이다. 16:9 슬라이드 위에 실제 BPMN 모델(bpmn-js)을 렌더링하고 토큰을 얹어 실행 의미를 관찰한다.

확정된 아이덴티티(v3)는 **빛과 그림자의 반전**이다:

- **표지 = 어둠 속의 빛.** 따뜻한 어둠(`#191816`)을 가르는 수직 광선 하나, 그 곁의 조용한 활자.
- **본문 = 빛(종이) 위의 그림자.** 따뜻한 종이 그라디언트 위에, 좌측에서 고요히 드리우는 그림자. 그림자 안에 단원명(세로)과 복무 질문이 산다.

읽히는 순서는 고정이다: **① 메시지 → ② 도식·숫자(증거) → ③ 구조 요소(그림자·라벨) → ④ 배경(분위기).** 화면의 모든 요소는 의미를 나른다 — 의미 없는 장식(배지·바·칩·그림자박스)은 금지.

**핵심 특징**
- **메시지 우선**: 각 장표의 압축된 핵심 문장이 화면에서 가장 크고 먼저 읽힌다(38px/680, 상단). 별도의 하단 "핵심 문장" 배지는 없다 — 메시지가 곧 제목.
- **화면 텍스트 예산**: 슬라이드당 대략 30~60단어. 설명은 `aside.notes`(강사 대본)로.
- **도식은 증거**: bpmn-js 렌더를 종이 위에 투명하게(흰 면 제거) — 잉크로 그린 듯. 큰 모델은 설명 구간으로 `focus()` 줌.
- **색은 의미가 있을 때만**: 강조·토큰 `#457075`, 오류·교착 테라코타, 생성됨 녹색, 대기 황토.
- **이모지·픽토그램 금지**, **강사·운영 컨텍스트 화면 노출 금지**(COURSE.md §4).
- Pretendard Variable 벤더링 — 발표 PC 무관 동일 렌더.

> **구현 매핑**: `{token}`은 `assets/css/theme.css :root` CSS 변수와 대응. 색 변경은 `:root` 한 줄.

## Slide Content Principles — 강의 전달 원칙

새 장표를 만들거나 리뷰할 때의 판정 기준.

1. **제목/메시지** — 그 장표의 압축된 메시지가 화면 최상단·최대 크기. 목차만 읽어도 내용이 보이게. 한 슬라이드 한 주제.
2. **강조 1~2개** — 크기·무게·의미색의 대비로. 중첩 금지.
3. **본문 최소** — 블록 ≤ 3, 30~60단어. 대사는 notes로.
4. **시각 자료가 주인공** — 그림이 보여 주는 것을 글로 반복하지 않는다.
5. **사례 먼저** — 개념은 온보딩 장면에서 꺼낸다. 장면 서술은 notes와 메시지가 겸한다.
6. **단계 노출** — 말하는 순서대로 fragment. 토큰 재생은 fragment와 동기화(STEPS).
7. **메시지 한 줄** — 요약이 아니라 일반화된 교훈. (v3: 하단이 아니라 최상단)
8. **오해 교정** — `caveat`(테라코타 좌측 룰 한 줄)로 화면에 명시.
9. **출처** — 사실 주장에 `slide__cite`(우하단). verification-notes와 일치 의무.
10. **용어 일관성** — COURSE.md §7 사전.
11. **연결·회수** — `data-plan`(비가시)·inventory가 추적. 교시 회수는 여섯 질문에서.

### 자가 점검 체크리스트
1. 메시지만 읽어도 이 장표가 남기려는 것이 보이는가?
2. 아이디어 하나, 본문 블록 3개 이하, 30~60단어인가?
3. 3초 안에 눈이 가는 강조가 1~2개뿐인가?
4. 도식이 주장하고 텍스트가 안내하는가?
5. fragment 순서 = 말하는 순서인가?
6. 사실 주장에 출처가 있고 verification-notes와 일치하는가?
7. 흔한 오해를 화면에 명시했는가? (해당 시)
8. 무의미한 장식이 하나도 없는가?
9. 강사·운영 컨텍스트가 화면에 없는가?
10. 겹침·과밀 없이 1280×720에 들어가는가?

## Layout
- 캔버스 **1280×720 고정**, reveal 스케일.
- 본문 구조: `.slide__shade`(좌측 그림자) + `.slide__label`(세로 단원명, 그림자 안) + `.slide__q`(복무 질문, accent) + `.slide__inner`(left 150 / right 84, 하단 패딩 44).
- inner 세로: **메시지(72px 상단 여백) → 콘텐츠(flex) → support(절대 bottom 60) → cite(bottom 34)**.
- 표지: 광선 left 46%, 활자 좌하단, 세로 라벨은 광선 오른쪽.

## Components

### `cover` — 표지(어둠 속의 빛)
`cover__slit`(수직 광선 + 글로우) · `cover__kicker` · `cover__vert`(세로 "신입사원 온보딩") · `cover__title`(작은 활자, b가 강조 줄) · `cover__corp`. 표지의 주인공은 빛이고, 활자는 조용하다.

### `slide` — 본문(빛 위의 그림자)
`slide__shade` + `slide__label` + `slide__q` + `slide__inner`. 그림자는 장식이 아니라 **내비게이션**(단원·복무 질문의 자리)이며 표지의 어둠과 짝을 이룬다.

### `slide__msg` — 메시지
화면 최대 요소. 압축된 교훈 한두 줄. 이것이 곧 제목이다.

### `slide__support` / `slide__cite`
하단 보조 근거 한 줄(mute) / 출처(모노, 흐림).

### 구도 컴포넌트 — 같은 구도 반복 금지
- **`vs`** 대형 매핑: 30px 용어쌍(OS=ink-soft ↔ BPMN=accent) + 중앙 `↔` + 카테고리 캡스. fragment로 행 단위 등장.
- **`relist` + `facet-row`** 목록 재배열: 상단 목록 스트립(캡 + 중점 구분 항목) ↔ 전폭 다이어그램 대비 + 하단 구조 단어 행(fragment, 현재 단어만 accent).
- **`rows`** 행 원장: 좌 용어(25px/740, 강조 행만 `--hi`=accent) + 우 한 줄 설명(mute)의 반복 구조. 도구·기준 비교에 사용, fragment로 행 단위 등장.
- **`bpmn` 전면**: 종이 위 투명 렌더(흰 면 CSS 제거). `bpmn__cap`은 mute 캡스 한 줄. 하단 `hero-foot`(persona 점 + 힌트).
- **`cols--jumbo`**: 좌 bpmn + 우 132px accent 숫자(`fade-in-then-out`으로 같은 자리 교체 — 1→3→1→0).
- **`dead-strip`**: 도식 아래 한 줄 색 원장(`ok`/`no`/`wt`).
- **`flow`**: 가로 4단계 타임라인 — 50px thin 모노 번호(ok 녹/중립 mute/bad 테라코타).
- **`caveat`**: 테라코타 2px 좌측 룰의 오해 교정 한 줄.

### BPMN 마커·토큰
- 마커: `highlight-flow`·`active-el`=accent(+옅은 fill), `dead-el`=테라코타 점선(+옅은 fill), `dim-el`=0.3.
- **`tok`**: accent 원(26px), **무라벨 점** — 개수는 jumbo 숫자가 말한다. 요소 **우상단**에 얹어 라벨을 가리지 않는다. `tok--wait`(회갈색), `tok--ghost`(테라코타 점선 = 존재하지 않는 토큰).

## Do's and Don'ts

### Do
- 메시지를 먼저 쓰고, 그것을 가장 크게 놓고, 나머지를 그 아래로 물린다.
- 도식에 최대 면적. 설명 구간으로 줌. 존재하지 않는 것은 점선.
- 색이 나오면 "무슨 의미인가"를 답할 수 있어야 한다.
- 그림자·광선은 각 1개, 항상 같은 자리 — 정체성이지 장식이 아니다.

### Don't
- 이모지·픽토그램, 무의미한 바·배지·칩 금지.
- 어두운 면·새 표면 색 추가 금지(어둠은 표지의 것).
- 강조·오류·성공 계열 색을 서로 침범하지 않는다.
- 본문 텍스트로 화면을 채우지 않는다 — 넘치면 notes로.
- 강사·운영 컨텍스트(교시·시간·설계안 번호·대상·전제) 화면 노출 금지.

## Responsive & Export
- 16:9 고정, reveal 스케일. 폰트·에셋 벤더링으로 오프라인·PDF 동일.
- 로컬 에셋은 `?v=N` 캐시 버스터를 사용(수정 시 N 증가).
- PDF: `?print-pdf` 또는 decktape. PPTX는 확정 후 별도 논의.

## Iteration Guide
1. 색은 `:root` 토큰만. 새 색은 의미 정의 없이 추가 금지.
2. 새 장표는 **shade+label+q → msg → 구도 컴포넌트 → support(+cite)** 골격 재사용. 구도는 내용 구조에 맞게 선택(vs/전면/jumbo/dead-strip/flow), 같은 구도 연속 반복 금지.
3. 토큰 재생은 deck.js `STEPS` — fragment 수 = 배열 길이 − 1, 요소 id는 grep으로 확인.
4. 장표 완성 전 자가 점검 10문항 + slide-reviewer 판정.
5. theme.css/deck.js 수정 시 index.html의 `?v=` 버전을 올린다.
